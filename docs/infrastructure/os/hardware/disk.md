# disk 

**磁盘页（Disk Page）**，通常在数据库（如 MySQL、Oracle）和操作系统（OS）的语境中被提及，它是**磁盘和内存之间进行数据交换（I/O 操作）的最小逻辑单位**。

简单来说，当程序想从磁盘读取数据时，哪怕只想读一个字节，系统也不会只读这一个字节，而是会把包含这个字节的整整一“页”数据全部加载进内存。


---

## 1. 为什么需要“页”这个概念？

如果每次读写都直接对应磁盘的物理位置，效率会极低。

* **减少 I/O 次数：** 磁盘访问（尤其是机械硬盘的寻道，或者是固态硬盘的擦写）是非常慢的。通过一次性读取一整页，可以大幅减少与磁盘交互的次数。
* **局部性原理（Principle of Locality）：** 计算机科学家发现，程序在访问一段数据时，它周围的数据在不久的将来也很大概率会被访问。把一整页数据一起读进内存，就是为了“未雨绸缪”。

---

## 2. 磁盘页、扇区、操作系统页的关系

这三者很容易混淆，它们其实处于不同的层级，大小也不同：

| 概念 | 属于谁 | 典型大小 | 角色 |
| --- | --- | --- | --- |
| **扇区 (Sector)** | 物理磁盘 | 512 字节 或 4KB | **硬件层面**的最小读写单位。 |
| **系统页 (OS Page)** | 操作系统 | 4KB | **操作系统**管理内存和磁盘交互的最小单位（通常是 8 个 512 字节的扇区组成 1 个 OS 页）。 |
| **数据库页 (Database Page)** | 数据库 (如 MySQL InnoDB) | **16KB** (可调) | **数据库管理系统**为了提高效率，在 OS 页之上封装的更大单位（1 个 InnoDB 页 = 4 个 OS 页）。 |



```shell
# 查看os pagesize 
getconf PAGESIZE 
```

```sql 
SHOW global VARIABLES LIKE 'innodb_page_size'; -- 全局 服务器  查看数据库配置的页大小
```

## 3. 在数据库（以 MySQL InnoDB 为例）中的应用

当你听到“磁盘页”时，绝大多数场景是在讨论数据库调优或索引原理（比如 B+ 树）。

在 MySQL 的 InnoDB 存储引擎中，一个页的大小默认是 **16KB**。

* **B+ 树的节点：** B+ 树的每一个节点（包括叶子节点和非叶子节点）在磁盘上都是一个 16KB 的页。
* **页内结构：** 一个 16KB 的页里面，不仅存你插入的行数据（User Records），还包含页头（Page Header，存一些元数据）、页尾（File Trailer，用于校验数据是否损坏）以及页目录（Page Directory，用于在页内快速通过二分法查找记录）。
